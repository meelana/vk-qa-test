Задание сделано с использованием фреймворков

 junit, selenide, паттерн PageObject , gradle

ЗАПУСК

Обязательно Java 20 , тк при создании проекта в Intellij Idea написано, что gradle не работает с Java 22, а только работает с Java 20

Для запуска всех тестов нужно выполнить команду 

 gradlew cleanTest test

Чтобы выполнить только какой-то один тест из Test1, Test2, Test3, Test4

 gradlew cleanTest test  --tests Test1

Чтобы выполнить несколько тестов из Test1, Test2, Test3, Test4,  можно выбрать нужные и написать через пробел

 gradlew cleanTest test  --tests Test1  --tests Test3

Фреймворк allure2 не сделан,  без него задание было очень сложным

ЗАДАНИЕ

На Википедии на каждой странице есть форма поиска. К ней предъявлены следующие бизнес-требования:
1. При вводе первые саджесты начинаются на поисковый запрос. Далее на усмотрение поискового движка. Поисковый запрос выделяется жирным шрифтом.
2. Переход из саджеста по поисковой фразе идет в точности на ее страницу. 
3. При вводе поискового запроса и нажатии кнопки поиск (лупа справа) - переходим на первую подсказку саджеста, если он был. Если не было - на страницу поиска. (Иван - переходит на страницу Иван, Иваннннн переходит на поиск с вхождением Иваннннн)
4. Снизу всегда есть подсказка: "Поиск страниц содержащих", при клике на нее выполняется переход на страницу поиска.

Требуется написать набор автотестов, которые верифицируют данные требования. Число и содержание тест-кейсов на ваше усмотрение. При нехватке требований - выполнить допущение, исходя из общепринятых знаний и здравого смысла. Об этом сделать комментарий в коде.
В тестах придерживаться паттерна PageObject. Тесты должны иметь возможность запускаться в параллель. В каждом тесте должны быть написаны комментарии. При запуске тестов c консоли в конце прогона должен выводиться отчет о прохождении тестов.
Если какой-то тест не получится автоматизировать, его необходимо записать и сделать так, чтобы он всегда падал с причиной "Не автоматизирован". В комментарии написать почему именно у вас не получилось это сделать.

Будет плюсом (необязательно):
Подключение системы отчетности allure2 (и написание документации как сгенерировать и просмотреть отчет).

В поле ответа вставь ссылку на репозиторий на github.
В корне должен лежать readme файл с описанием того, как запустить один тест, несколько тестов, все тесты в консольном режиме. Проект должен быть работоспособен сразу после checkout командой из readme.
